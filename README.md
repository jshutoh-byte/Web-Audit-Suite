# 🛠️ Web Audit Suite v3.5
### 〜AIによる「技術的負債」と「業者の誠実さ」の可視化ツール〜

## 🌟 概要
本ツールは、Webサイトの表面的なデザインではなく、その「内部構造（コードの筆跡）」から**技術的負債**と**セキュリティリスク**を客観的に判定する診断スイートです。

特に、専門知識のない企業の担当者が「なぜ自社サイトが更新しづらいのか？」「この保守費用は妥当なのか？」という疑問に対し、AI（LLM）の推論を用いて明確な回答と改善策を提示します。

---

## 🚀 主な機能

### 1. 🔍 精密構造監査 (Security & Quality Mode)
**【コンサルタント・監査人視点】**
セマンティックなマークアップ、アクセシビリティ（alt属性）、そしてセキュリティリスク（Attack Surface）を論理的に査定します。
* **用途:** 企業の経営層やIT部門への公式な報告・改善提案。
* **特徴:** ビジネスリスクとしての「技術的負債」を数値化します。

### 2. 💀 欠陥建築Gメン (Roast Strict Mode)
**【現場監督・技術者視点】**
「動けばいい」という甘えを許さず、業者の「手抜き」や「囲い込み（集金システム）」を暴き出します。
* **用途:** 納品物の品質チェック、不誠実なベンダーへの是正勧告。
* **特徴:** 負債による将来的な損失を「被害想定額」として算出するエンタメ性を備えています。

### 3. 🖼️ Asset Inspector
ページ内の全画像を一覧抽出し、表示速度を阻害する巨大画像や、SEOを妨げるファイル名設定を特定します。

## 🧠 AIモデルによる診断精度の違い（重要）
本ツールは使用するAI（LLM）によって、診断の「深さ」が変化するセカンドオピニオン機能を備えています。

| AIモデル | 診断傾向 | 推奨用途 |
| :--- | :--- | :--- |
| **Gemini Pro (推奨)** | **「冷徹なベテラン監督」**。コードの歴史的背景（ASP.NET Web Forms等のレガシー技術）を敏感に察知し、本質的なリスクを指摘します。 | 厳格な監査、本質的な改善提案 |
| **ChatGPT (GPT-4o)** | **「優秀な新人エンジニア」**。表層的なエラーには敏感ですが、レガシー技術に対してはやや楽観的です。※v3.5では強制制約により厳格化。 | 一般的な品質チェック |

## 📱 スマートフォンでの使用方法
スマートフォンでも「ブックマークの編集機能」を利用することで、本ツールを使用可能です。

### 1. 準備（登録手順）
1. 本ページをブラウザで開き、適当なページ（このページ等）を**ブックマークに追加**します。
2. 追加したブックマークの**「編集」**を開きます。
3. 名前を「Gメン診断」等に変更し、**URL欄の中身をすべて削除**します。
4. 本ページの「💀 欠陥建築Gメン」ボタンを長押しして**「リンクをコピー」**し、手順3のURL欄に貼り付けて保存します。

### 2. 実行方法
1. 診断したいWebサイトを開きます。
2. アドレスバー（URL入力欄）に、登録した名前「Gメン診断」を入力します。
3. 検索候補に表示されるブックマークをタップすると、診断スクリプトが実行されます。
4. 「証拠確保完了」と表示されたら、AIアプリ（Gemini等）を開いてペーストしてください。

## 🛠️ 技術スタック
* **Frontend:** HTML5, CSS3, JavaScript (Vanilla JS)
* **Mechanism:** ブックマークレットによるDOM解析・プロンプトエンジニアリング
* **Heuristics:**
    * ASP.NET Web Forms (`__VIEWSTATE`) 自動検知
    * Table Layout / fontタグ等のレガシー構造スキャン
    * セキュリティリスク（非暗号化通信、過剰iframe等）の嗅ぎ出し（Sniffing）
